# Mission Editor

Tauri v2 desktop app for editing mission bundle JSON files used by the DragonBlockCRevisited Minecraft Forge 1.7.10 mod.

## Tech Stack

- **Frontend:** React 19 + TypeScript (strict mode) + Vite 7
- **Backend:** Tauri v2 (Rust, minimal: only plugin init, no custom commands)
- **Package Manager:** pnpm
- **State:** React hooks only, no external state library. `useHistory` hook owns bundle state as single source of truth with debounced undo/redo.

## Build & Run

```bash
pnpm install
pnpm tauri dev       # Run the app in dev mode (Vite on port 1420)
pnpm build           # TypeScript check + Vite production build (frontend only)
pnpm tauri build     # Full production build with Tauri (outputs installer)
pnpm gen-lang        # Regenerate translation files from mod lang files + MC jar
```

Type-check only (no emit): `npx tsc --noEmit`

## Project Structure

```
src/
  App.tsx             # Root component, toolbar, all top-level state and handlers
  App.css             # All styles (single file, organized with #region blocks)
  components/         # React components (Sidebar, MissionEditor, ObjectiveEditor, etc.)
  utils/              # Pure logic: file I/O, parsing, validation, i18n, history, settings
  types/mission.ts    # Core types: Mission, MissionBundle, Objective, RewardChoice
public/lang/          # Generated JSON translation files (11 languages)
src-tauri/            # Rust backend (plugin registration only)
gen-translations.cjs  # Node script: extracts lang keys from mod resources + vanilla MC jar
```

## Data Model

Mission bundles are JSON files matching the Gson-serialized `JRMCoreMsnBundle` Java class. Key structures:

- **MissionBundle**: `{ name, version, authors, mods, desc, missions[], settings }`
- **Mission**: Has parallel arrays indexed by property variant: `props[], title[], subtitle[], description[], align[], objectives[][], rewards[][]`
- **Objectives**: Semicolon-delimited strings like `"kill;NZombie;H100;A10;M1"`. Parsed by `utils/objectives.ts`.
- **Rewards**: Format `"component1||component2;buttonName;nextMissionId"`. Components like `"tp!fix!150"`, `"item!mod:name,5"`. Parsed by `utils/rewards.ts`.

Items without a `modid:` prefix auto-resolve to `minecraft:` in the mod's mission system.

## I18n

Two separate translation systems:

1. **App UI strings** (`utils/i18n.ts`): Uses `i18next` + `react-i18next`. Initialized in `i18n.ts`, imported in `main.tsx`. Components use `const { t } = useTranslation()` hook. Non-component code can use `i18next.t()` directly. Interpolation uses `{{named}}` placeholders. 6 languages (EN complete, others partial with EN fallback). Language switching via `i18n.changeLanguage()`.
2. **Mod translations** (`utils/translations.ts`): Runtime-loaded from `public/lang/*.json`. Generated by `gen-translations.cjs` from the mod's `.lang` files and vanilla Minecraft jar. Used for autocomplete hints and translation previews.

`gen-translations.cjs` also generates `utils/languages.ts` (available language list) and `utils/registry.ts` (entity/item ID lists).

## Key Patterns

- **Single source of truth**: The `useHistory` hook in App.tsx owns all bundle state. No dual-state sync. All mutations go through `setBundles()` which the hook provides.
- **Debounced undo**: Rapid edits (typing) batch into one undo entry (500ms). Structural changes (add/delete/reorder) commit immediately via `setBundles(value, true)`.
- **Validation**: `utils/validation.ts` runs on every bundle change (via `useMemo`). Checks duplicate IDs, array length mismatches, orphaned missions, reward references, objective fields.
- **File validation**: `validateBundleStructure()` in `utils/files.ts` checks JSON shape before accepting opened files. Errors shown via toast.
- **Canvas flow graph**: `FlowGraph.tsx` renders mission links on a `<canvas>` with pan/zoom. Orphaned nodes get dashed orange borders, backward edges drawn in warning color.
- **Pointer-based drag-and-drop**: Sidebar mission reordering uses pointer events (not HTML5 drag API) for reliability.

## Architecture Notes

- All file I/O uses Tauri plugins (`@tauri-apps/plugin-fs`, `@tauri-apps/plugin-dialog`). No custom Rust commands.
- All styles in `App.css` using CSS custom properties for theming. Dark/light theme support.
- Components read CSS variables at runtime (flow graph canvas reads `--accent`, `--bg-panel`, etc. from `getComputedStyle`).
- `@SideOnly` is irrelevant here: this is a standalone desktop editor, not a Minecraft mod. The bundle JSON files it produces are consumed by the mod.
